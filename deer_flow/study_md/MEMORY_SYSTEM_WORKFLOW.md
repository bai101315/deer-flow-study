# DeerFlow 项目记忆系统工作流程

本文档详细描述了 DeerFlow 项目中记忆系统的读取、加载和更新过程，涵盖了主要模块和功能。

---

## 1. 系统架构概览

记忆系统由以下几个核心模块组成：

- **`storage.py`**: 提供记忆的存储和加载功能，支持文件系统存储。
- **`updater.py`**: 负责记忆的读取、写入和更新。
- **`queue.py`**: 实现了带防抖机制的记忆更新队列。
- **`prompt.py`**: 定义了记忆更新的提示模板。
- **配置文件 (`config.yaml`)**: 定义了模型、工具和记忆系统的相关配置。

---

## 2. 记忆读取与加载流程

### 2.1 文件存储模块（`storage.py`）

#### 功能
- **存储结构**: 记忆数据以 JSON 格式存储在文件中。
- **缓存机制**: 使用内存缓存减少文件读取次数。
- **路径管理**: 根据代理名称动态生成存储路径。

#### 核心方法
1. **`load(agent_name: str | None)`**:
   - 从文件中加载指定代理的记忆数据。
   - 如果文件不存在，返回空的记忆结构。

2. **`reload(agent_name: str | None)`**:
   - 强制重新加载记忆数据，忽略缓存。

3. **`save(memory_data: dict, agent_name: str | None)`**:
   - 将记忆数据保存到文件。
   - 验证代理名称是否合法，防止路径遍历攻击。

4. **`create_empty_memory()`**:
   - 创建一个空的记忆数据结构，包含用户上下文、历史记录和事实。

---

### 2.2 记忆更新模块（`updater.py`）

#### 功能
- 提供记忆数据的读取、清除和导入功能。
- 支持从对话中提取信息并更新记忆。

#### 核心方法
1. **`get_memory_data(agent_name: str | None)`**:
   - 获取当前记忆数据。

2. **`reload_memory_data(agent_name: str | None)`**:
   - 强制重新加载记忆数据。

3. **`import_memory_data(memory_data: dict, agent_name: str | None)`**:
   - 导入新的记忆数据并保存。

4. **`clear_memory_data(agent_name: str | None)`**:
   - 清空记忆数据并保存为空结构。

---

### 2.3 记忆更新队列模块（`queue.py`）

#### 功能
- 实现了带防抖机制的记忆更新队列。
- 支持批量处理多个对话上下文。

#### 核心方法
1. **`add(thread_id: str, messages: list, ...)`**:
   - 将对话上下文添加到更新队列。
   - 如果队列中已存在相同线程的上下文，则合并更新。

2. **`_reset_timer()`**:
   - 重置防抖定时器，在指定时间后处理队列。

3. **`_process_queue()`**:
   - 处理队列中的所有上下文，更新记忆数据。

---

### 2.4 提示模板模块（`prompt.py`）

#### 功能
- 定义了记忆更新的提示模板。
- 提供对话内容的格式化工具。

#### 核心内容
1. **`MEMORY_UPDATE_PROMPT`**:
   - 提示模板，指导系统如何从对话中提取信息并更新记忆。

2. **`format_conversation_for_update(conversation: list)`**:
   - 格式化对话内容，便于后续处理。

---

## 3. 配置文件（`config.yaml`）

#### 关键配置
1. **模型配置**:
   - 定义了多个模型的名称、API 密钥、最大 token 数等。
   - 示例：
     ```yaml
     - name: minimax-m2.7
       display_name: MiniMax M2.7
       api_key: sk-xxxx
       max_tokens: 4096
     ```

2. **工具配置**:
   - 定义了可用工具及其分组。
   - 示例：
     ```yaml
     tools:
     - name: web_search
       group: web
     ```

3. **记忆系统配置**:
   - 是否启用记忆更新队列。
   - 示例：
     ```yaml
     token_usage:
       enabled: false
     ```

---

## 4. 工作流程总结

1. **初始化**:
   - 加载配置文件，初始化存储模块和更新队列。

2. **记忆读取**:
   - 调用 `storage.py` 的 `load` 方法，从文件中加载记忆数据。

3. **记忆更新**:
   - 对话完成后，调用 `queue.py` 的 `add` 方法，将对话上下文加入队列。
   - 定时器触发后，调用 `updater.py` 的方法更新记忆。

4. **记忆保存**:
   - 调用 `storage.py` 的 `save` 方法，将更新后的记忆数据保存到文件。

---

## 5. 注意事项

- **路径安全**: 确保代理名称合法，防止路径遍历攻击。
- **防抖机制**: 合理配置防抖时间，避免频繁更新。
- **配置管理**: 确保 `config.yaml` 中的路径和 API 密钥正确无误。

---

以上是 DeerFlow 项目记忆系统的详细工作流程。如需进一步优化或调试，请参考各模块的具体实现。
